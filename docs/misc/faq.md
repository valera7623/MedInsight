# FAQ

Частые вопросы по MedInsight.

## Общие

### Что такое MedInsight?

Платформа для хранения карточек пациентов, анализа медицинских документов, просмотра DICOM и AI-прогнозов рисков.

### Нужен ли интернет?

Да, для работы веб-интерфейса и GPT-прогнозов. Документы и DICOM обрабатываются на вашем сервере.

### Поддерживается ли несколько клиник?

Да, через **мультитенантность** — каждая клиника имеет свой subdomain.

## Вход и роли

### Забыл пароль

Используйте «Забыли пароль» на странице входа (если включён SMTP). Иначе обратитесь к администратору.

### Не вижу раздел «Админ»

Доступен только роли **admin**.

### Чем исследователь отличается от наблюдателя?

**Исследователь** видит анонимизированные данные. **Наблюдатель** — полные, но без права редактирования.

## Документы

### Какие форматы поддерживаются?

PDF и DOCX.

### Долго висит статус «processing»

Обычно 1–2 минуты. Если >10 мин — проверьте Celery worker (администратор).

### Безопасно ли хранить выписки?

Файлы шифруются **age** на сервере. Доступ по RBAC и tenant isolation.

## DICOM

### Какой максимальный размер файла?

По умолчанию 500 МБ (`DICOM_MAX_FILE_SIZE_MB`).

### Можно загрузить папку с серии?

Пока — один `.dcm` за раз. ZIP-поддержка в roadmap.

## Прогнозы

### Насколько точны прогнозы?

Это **поддержка решения**, не диагноз. Точность зависит от полноты документов и модели GPT.

### Что если GPT недоступен?

Работает rule-based fallback без текстового пояснения.

### Превышен лимит анализов

Обновите тариф на странице **Подписка** или установите `BILLING_ENABLED=false` (тест).

## Администрирование

### Как обновить систему?

Push в `main` → GitHub Actions деплой. Или `./deploy.sh production` на VPS.

### Как сделать бэкап?

См. [Резервное копирование](../admin-guide/backup.md). Автобэкап при `BACKUP_ENABLED=true`.

### Ошибка при удалении пользователя

Обновите до последней версии — исправлены FK constraints.

## Технические

### Где Swagger API?

`/docs` на работающем сервере.

### SQLite или PostgreSQL?

По умолчанию SQLite. PostgreSQL рекомендуется для высокой нагрузки.

### Как запустить документацию локально?

```bash
pip install -r requirements-docs.txt
mkdocs serve
```

### Где документация на проде?

| Язык | URL |
|------|-----|
| Русский | `https://fileguardian.com.ru/help/` |
| English | `https://fileguardian.com.ru/help/en/` |

Язык переключается в шапке документации или автоматически при первом визите (по языку браузера). API Swagger — `/docs`.
