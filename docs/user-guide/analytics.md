# Аналитика и дашборд

## Зачем нужен дашборд

Дашборд даёт обзор клиники: сколько пациентов и документов, какие диагнозы и лекарства встречаются чаще всего, есть ли DICOM-исследования.

## Открытие

После входа вы попадаете на **Дашборд** (`/`).

**Скриншот:** верхняя строка с ФИО пользователя; карточки «Пациентов», «Документов», «Диагнозов»; два графика Chart.js.

## Фильтр по отделению

В шапке дашборда выберите **отделение** в выпадающем списке или «Все отделения».

**Зачем:** смотреть статистику только по своему отделению или по всей клинике (если есть права).

## Блоки дашборда

### Сводные цифры

| Карточка | Содержание |
|----------|------------|
| Пациентов | Количество доступных пациентов |
| Документов | Загруженных и распознанных |
| Диагнозов | Уникальных диагнозов в данных |

### Графики

- **Топ-5 диагнозов** — столбчатая диаграмма.
- **Топ-5 лекарств** — столбчатая диаграмма.

Графики автоматически подстраиваются под **тёмную тему**.

### Прогнозы и риски

- число пациентов с **высоким риском**;
- графики рисков по отделениям и по месяцам;
- таблица пациентов с высоким риском.

Подробнее: [Прогнозы](predictions.md).

### Вебхуки

Блок для настройки **исходящих интеграций** (для продвинутых пользователей). Обычно настраивает администратор.

### Недавние пациенты

Быстрый переход к последним добавленным карточкам.

## DICOM в аналитике

На дашборде учитываются метаданные DICOM:

- количество исследований;
- распределение по **модальностям** (CT, MR…);
- распределение по **областям тела**.

## Экспорт данных

- **Excel** — списки пациентов, документов, прогнозов (кнопки на соответствующих страницах).
- **PDF** — отчёт по одному пациенту.

## Обновление данных

Дашборд обновляется при перезагрузке страницы. Новые документы и прогнозы появятся после парсинга/обработки Celery.
