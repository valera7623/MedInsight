# Прогнозы рисков (AI)

## Зачем нужны прогнозы

MedInsight анализирует данные пациента (документы, диагнозы, лекарства) и оценивает:

- **риск реадмиссии** — вероятность повторной госпитализации;
- **риск осложнений** — вероятность неблагоприятного исхода.

Это **поддержка решения врача**, а не замена клинического суждения.

## Кто может запускать прогноз

Врачи, зав. отделением, администраторы — роли с правом записи (`WRITE_ROLES`).

## Запуск прогноза

1. Откройте **карточку пациента**.
2. Убедитесь, что есть хотя бы один **распознанный документ** (`parsed`).
3. Нажмите **Запустить прогноз**.
4. Дождитесь завершения (обычно 10–60 секунд).

**Скриншот:** карточка пациента с кнопкой «Запустить прогноз» и индикатором «Обработка…».

## Результат

На карточке отображается:

- **вероятность** реадмиссии и осложнений (%);
- **уровень риска**: низкий / средний / высокий;
- **пояснение** от GPT (если доступен ProxyAPI);
- дата прогноза.

На **дашборде** пациенты с высоким риском попадают в отдельную таблицу.

## Лимиты тарифа

Количество анализов в месяц ограничено тарифом:

| Тариф | Лимит (пример) |
|-------|----------------|
| Freemium | 5 |
| Pro | 100 |
| Enterprise | без ограничений |

Текущее использование — на странице **Подписка** (`/subscription`).

При превышении лимита API вернёт ошибку **402 Payment Required**.

## Уведомления

При готовности прогноза:

- **WebSocket** — обновление в браузере (если открыт);
- **Email** — если настроен SMTP;
- **Telegram** — если аккаунт привязан.

## Валидация прогноза

Врач может **подтвердить или опровергнуть** прогноз (через API `validate-prediction`) — данные используются для улучшения системы.

## Если GPT недоступен

При отсутствии ключа ProxyAPI или сбое сети система использует **rule-based fallback** — упрощённый расчёт на основе правил, без GPT-пояснения.

## Безопасность

- Данные пациента не покидают ваш сервер, кроме запросов к ProxyAPI (если настроен).
- Прогнозы привязаны к **tenant_id** — другие клиники их не видят.
